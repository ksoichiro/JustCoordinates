# Design: Minecraft 26.1 Support

## Summary

Add Minecraft 26.1 support (Fabric, NeoForge, Forge) to JustCoordinates. MC 26.1 is the first version under the new versioning scheme, following 1.21.11.

## Key Version Changes

| Item | 1.21.11 | 26.1 |
|------|---------|------|
| Java | 21 | 25 |
| Gradle | 8.14.4 | 9.1.0+ |
| Fabric Loader | 0.17.3 | 0.18.5 |
| Fabric API | 0.141.3+1.21.11 | 0.144.3+26.1 |
| Fabric Loom | 1.13.6 (`fabric-loom`) | 1.15 (`net.fabricmc.fabric-loom`) |
| NeoForge | 21.11.38-beta | 26.1.0.1-beta |
| NeoForm | 1.21.11-20251209.172050 | 26.1-1 |
| ModDevGradle | 2.0.140 | 2.0.141 |
| Forge | 1.21.11-61.0.8 | 26.1-62.0.4 |

## Breaking Changes

### Build Infrastructure
- **Gradle 9.1.0+** required ‚Äî project-wide wrapper upgrade needed
- **Java 25** ‚Äî `foojay-resolver-convention` must support Java 25 provisioning; Gradle JVM itself needs Java 25 or compatible version
- **Fabric Loom plugin ID** changed: `fabric-loom` ‚Üí `net.fabricmc.fabric-loom`
- **Obfuscation removed** ‚Äî Fabric no longer needs remapping
- **Fabric dependency syntax**: `modImplementation` ‚Üí `implementation` (affects only `fabric/26.1/build.gradle`, not `fabric/base/`)
- **ModDevGradle**: 2.0.140 ‚Üí 2.0.141

### Minecraft API
- `GuiGraphics` ‚Üí `GuiGraphicsExtractor` (render method renames TBD via source JAR inspection)
  - If changes go beyond simple renames (e.g., different parameter types, removed methods), `common/26.1` architecture may need revision
- Fabric: `HudRenderCallback` removed ‚Üí `HudElementRegistry`
  - Registration pattern unknown; pre-implementation research step via Fabric API source JAR required

### Forge Build Plugin
- ForgeGradle compatibility with Gradle 9 and MC 26.1 is unconfirmed; will investigate during implementation

## Architecture

Follow established pattern: create version-specific directories for each loader.

### New Directories
- `props/26.1.properties`
- `common/26.1/` ‚Äî CoordinatesHudRenderer with `GuiGraphicsExtractor` API
- `fabric/26.1/` ‚Äî HudElementRegistry-based registration (own Java sources needed)
- `neoforge/26.1/` ‚Äî Updated event registration
- `forge/26.1/` ‚Äî Updated event registration

### props/26.1.properties

```properties
minecraft_version=26.1
java_version=25
fabric_loader_version=0.18.5
fabric_api_version=0.144.3+26.1
neoforge_version=26.1.0.1-beta
neoform_version=26.1-1
forge_version=26.1-62.0.4
enabled_platforms=fabric,neoforge,forge
```

Note: `pack_format` value for 26.1 is unknown; will determine from MC source or Forge MDK during implementation.

### Build Changes

**settings.gradle ‚Äî 3-way Loom plugin selection:**
- MC 1.16.5: `dev.architectury.loom` (existing)
- MC 1.17.1‚Äì1.21.11: `fabric-loom` (existing)
- MC 26.1+: `net.fabricmc.fabric-loom` (new)

**gradle.properties:**
- Keep `fabric_loom_version=1.13.6` for pre-26.1 versions
- Add `fabric_loom_26_version=1.15` for 26.1+ (different plugin ID, not backward compatible)
- `moddevgradle_version` ‚Üí 2.0.141

**Gradle wrapper:** Upgrade to 9.1.0+

### Source Sharing
- `common/shared/` (JustCoordinates.java) ‚Äî no changes expected
- `fabric/base/` ‚Äî cannot be shared with 26.1 due to HudRenderCallback removal; 26.1 needs own Java source
- `neoforge/base/` ‚Äî shareability TBD (depends on whether RegisterGuiLayersEvent still exists)

### Mod Metadata
- `fabric/26.1/src/main/resources/fabric.mod.json` ‚Äî update version ranges
- `neoforge/26.1/src/main/resources/META-INF/neoforge.mods.toml` ‚Äî update version ranges
- `forge/26.1/src/main/resources/META-INF/mods.toml` ‚Äî update version ranges
- `forge/26.1/src/main/resources/pack.mcmeta` ‚Äî update pack_format

## Implementation Order

1. **Gradle 9 upgrade + existing build verification**
   - Upgrade wrapper to 9.x
   - Update plugin versions in `gradle.properties`
   - Update `settings.gradle` for 3-way Loom plugin selection
   - Test build for 1.21.11 (latest existing version)
   - **Decision point:** If Gradle 9 breaks older versions, evaluate:
     a. Fix compatibility issues for affected versions
     b. Defer older version support (continue with 26.1 work, fix older versions later)
     c. Reconsider approach (separate wrapper, etc.)
2. `props/26.1.properties`
3. `common/26.1` ‚Äî copy from 1.21.11, build, research and fix API renames via source JAR
   - If API changes are non-trivial, reassess common code architecture before proceeding
4. `fabric/26.1` ‚Äî research HudElementRegistry via Fabric API source JAR, implement, build
5. `neoforge/26.1` ‚Äî build, fix API changes
6. `forge/26.1` ‚Äî determine build plugin compatibility, build, fix API changes
7. Final build verification across representative versions (1.21.11 + 26.1 minimum)

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| Gradle 9 breaks older MC version builds | High ‚Äî blocks all development | Decision point at Step 1; fix or defer |
| Architectury Loom (1.16.5) incompatible with Gradle 9 | Medium ‚Äî only affects 1.16.5 | May need to pin 1.16.5 to older Gradle or drop support |
| ForgeGradle for older Forge incompatible with Gradle 9 | Medium | Investigate newer ForgeGradle versions or alternatives |
| `GuiGraphicsExtractor` changes more than rename | Medium ‚Äî requires common code rework | Research in Step 3 before proceeding |
| `HudElementRegistry` API significantly different | Low ‚Äî only affects Fabric entry point | Research in Step 4; contained to fabric/26.1 |
| Forge 26.1 build plugin unknown | Medium | Research during Step 6; may need new plugin |
| `foojay-resolver-convention` doesn't support Java 25 | Low | Update plugin version or configure toolchain manually |
