# Minecraft 26.1 Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Minecraft 26.1 support with Fabric, NeoForge, and Forge loaders.

**Architecture:** Copy 1.21.11 directories as base, upgrade Gradle to 9.x, update build scripts for new Fabric Loom plugin ID, then iteratively fix API changes discovered during builds. MC 26.1 removes obfuscation and renames several GUI classes.

**Tech Stack:** Java 25, Gradle 9.1.0+, Fabric Loom 1.15, ModDevGradle 2.0.141, ForgeGradle (TBD), NeoForge 26.1.0.1-beta, Forge 26.1-62.0.4

**Spec:** `docs/superpowers/specs/2026-03-28-minecraft-26.1-support-design.md`

## Task 5: Create fabric/26.1

**Files:**
- Create: `fabric/26.1/build.gradle`
- Create: `fabric/26.1/src/main/resources/fabric.mod.json`
- Create: `fabric/26.1/src/main/java/com/justcoordinates/fabric/JustCoordinatesFabric.java`

- [ ] **Step 1: Create `fabric/26.1/build.gradle`**

Key differences from 1.21.11:
- Plugin ID: `net.fabricmc.fabric-loom` (not `fabric-loom`)
- `modImplementation` → `implementation` (obfuscation removed)
- No `remapJar` (remap not needed)

```groovy
plugins {
    id 'net.fabricmc.fabric-loom'
}

configurations {
    commonJava {
        canBeResolved = true
    }
    commonResources {
        canBeResolved = true
    }
}

dependencies {
    minecraft "com.mojang:minecraft:${minecraft_version}"
    mappings loom.officialMojangMappings()
    implementation "net.fabricmc:fabric-loader:${fabric_loader_version}"
    implementation "net.fabricmc.fabric-api:fabric-api:${fabric_api_version}"

    compileOnly project(':common')

    commonJava project(path: ':common', configuration: 'commonJava')
    commonResources project(path: ':common', configuration: 'commonResources')
}

sourceSets.main {
    java {
        srcDir rootProject.file('common/shared/src/main/java')
    }
}

tasks.named('compileJava') {
    dependsOn configurations.commonJava
    source configurations.commonJava
}

tasks.named('processResources') {
    dependsOn configurations.commonResources
    from configurations.commonResources

    filesMatching('fabric.mod.json') {
        expand(
            version: mod_version,
            minecraft_version: minecraft_version,
            fabric_loader_version: fabric_loader_version,
        )
    }
}
```

Note: This does NOT include `fabric/base` srcDir because `HudRenderCallback` was removed. 26.1 needs its own Java entry point.
